# Основная идея движка
Разработчик должен создавать игры, а не решать архитектурные проблемы движка.

> - **DX (Developer Experience)** — главный приоритет архитектора движка.
> - **UX (User Experience)** — главный приоритет разработчика игры.

Только такой подход позволяет быстро создавать действительно хорошие игры.
# Описание
Движок создан для быстрой разработки проектов любого масштаба, однако основной фокус направлен на инди-разработку. Приоритет этого движка – создание удобного DX, который минимизирует основные проблемы современных игровых движков. Он оптимизирован для командной работы и предоставляет профессиональные инструменты для работы с нодами, Git и CI/CD.
Архитектура движка позволяет удобно разделять зоны ответственности между участниками команды, минимизируя конфликты контроля версий в процессе разработки.

Движок построен на базе .NET 10 и изначально поддерживает:
- многопоточность и параллельные вычисления;
- hot reload скриптов и ассетов;
- data-oriented архитектурный подход;
- реактивное обновление игровых данных во время игры;
- модификаторы данных;
- разделение пространств;
- удобную систему для внедрения зависимостей; 
- и многое-многое другое...
# Архитектура
Движок предлагает новый подход к разработке, объединяющий преимущества классического ООП, КОП и ECS-подходов: **EDL**.

---

# EDL
> **`Entity`** — пустой контейнер с метаданными.
>
> **`Data`** — структура реактивных данных.
>
> **`Logic`** — легковесный класс логики, содержащий только ссылку на `Entity`.
